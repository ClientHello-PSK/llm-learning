# 知识点 04：vLLM DP 架构与启动模式

> 核心问题：vLLM 怎么启动多个 DP rank？三种启动模式的区别？
> 源码位置：`vllm/config/parallel.py`、`vllm/entrypoints/`

---

## 1. vLLM 的三种 DP 启动模式

| 模式 | 命令行 | 适用 | DP 通信 |
|------|--------|------|---------|
| **MP（multiprocessing）** | `--data-parallel-size N` 默认 | 单机多卡 | 共享内存 |
| **Ray** | `--data-parallel-size N --distributed-executor-backend ray` | 多机 | Ray |
| **External** | 每个 DP rank 独立进程 + `VLLM_DP_RANK` | K8s/外部 LB | 无（外部） |

---

## 2. 模式一：MP（默认）

### 2.1 启动方式

```bash
vllm serve meta-llama/Llama-3-70B \
    --tensor-parallel-size 2 \
    --data-parallel-size 4
```

含义：

```
8 卡：
  TP=2 × DP=4 = 8 卡
  4 个 DP rank，每个 rank 内 TP=2
```

### 2.2 进程模型

```
vllm CLI 进程
   ↓ fork
DP rank 0 (主进程)
   ├─ EngineCore
   └─ Worker × TP=2
DP rank 1 (子进程)
   ├─ EngineCore
   └─ Worker × TP=2
DP rank 2 (子进程)
   ├─ EngineCore
   └─ Worker × TP=2
DP rank 3 (子进程)
   ├─ EngineCore
   └─ Worker × TP=2
```

每个 DP rank 有独立的 EngineCore + Worker，通过 `multiprocessing` 启动。

### 2.3 通信

- 同 DP rank 内：TP 用 NCCL all-reduce
- 跨 DP rank：通过 EngineCore 之间的 RPC（共享内存）
- 跨 DP rank（MoE 模型）：EP 用 NCCL all-to-all

源码：`vllm/config/parallel.py:144`

```python
data_parallel_backend: DataParallelBackend = "mp"
"""Backend to use for data parallel, either "mp" or "ray"."""
```

---

## 3. 模式二：Ray

### 3.1 启动方式

```bash
vllm serve meta-llama/Llama-3-70B \
    --tensor-parallel-size 2 \
    --data-parallel-size 4 \
    --distributed-executor-backend ray
```

### 3.2 进程模型

```
Ray Driver (主)
   ↓ ray.remote
Ray Worker 0 (DP rank 0)
   └─ TP workers × 2
Ray Worker 1 (DP rank 1)
   └─ TP workers × 2
...
```

每个 DP rank 是一个 Ray actor。

### 3.3 适用场景

- 跨节点部署
- 跟 Ray 生态集成（Ray Serve、Ray Data 等）
- 需要 Ray 的容错 / 调度能力

---

## 4. 模式三：External（多容器）

### 4.1 启动方式

每个 DP rank 启动一个独立的 vLLM 进程/容器：

```bash
# 容器 1
VLLM_DP_RANK=0 VLLM_DP_WORLD_SIZE=4 VLLM_DP_MASTER_IP=... \
vllm serve meta-llama/Llama-3-70B --tensor-parallel-size 2

# 容器 2
VLLM_DP_RANK=1 VLLM_DP_WORLD_SIZE=4 VLLM_DP_MASTER_IP=... \
vllm serve meta-llama/Llama-3-70B --tensor-parallel-size 2
...
```

源码：`vllm/config/parallel.py:842`

```python
self.data_parallel_size = envs.VLLM_DP_SIZE
```

### 4.2 进程模型

```
容器 1（独立的 vLLM 实例）
   └─ DP rank 0

容器 2（独立的 vLLM 实例）
   └─ DP rank 1
...
```

每个容器有自己的 API server、EngineCore。

### 4.3 外部 LB 路由

外部 LB（nginx、Kubernetes Service、HAProxy）路由请求：

```
client → LB → 容器 1 / 2 / 3 / 4
```

### 4.4 配置

源码：`vllm/config/parallel.py:146-152`

```python
data_parallel_external_lb: bool = False
"""Whether to use "external" DP LB mode. Applies only to online serving
and when data_parallel_size > 0. This is useful for a "one-pod-per-rank"
wide-EP setup in Kubernetes. Supported only for MoE deployments; non-MoE
models should use independent vLLM instances without --data-parallel-*
arguments. Set implicitly when --data-parallel-rank is provided explicitly
to vllm serve."""
```

**关键约束**：

- 只支持 MoE 部署（non-MoE 应该用独立 vLLM 实例）
- 用于 wide-EP 场景

## 5. Hybrid LB 模式

源码：`vllm/config/parallel.py:153-159`

```python
data_parallel_hybrid_lb: bool = False
"""Whether to use "hybrid" DP LB mode. Applies only to online serving
and when data_parallel_size > 0. Enables running an AsyncLLM
and API server on a "per-node" basis where vLLM load balances
between local data parallel ranks, but an external LB balances
between vLLM nodes/replicas. Set explicitly in conjunction with
--data-parallel-start-rank."""
```

### 5.1 适用场景

跨节点 wide-EP：

```
节点 1 (8 NPU, 内部 DP=2):
   rank 0-3 (DP rank 0)
   rank 4-7 (DP rank 1)
   vLLM 内部 LB 路由到 rank 0 / 1

节点 2 (8 NPU, 内部 DP=2):
   rank 8-11 (DP rank 2)
   rank 12-15 (DP rank 3)
   vLLM 内部 LB 路由到 rank 2 / 3

外部 LB:
   路由到节点 1 / 节点 2
```

### 5.2 与纯 external_lb 的区别

| 模式 | 节点内 | 节点间 |
|------|--------|--------|
| external_lb | 每 rank 独立 vLLM | 外部 LB |
| hybrid_lb | **节点内 vLLM 内部 LB** | 外部 LB |

hybrid_lb 节点内更高效（vLLM 内部 LB 比 HTTP LB 快）。

## 6. local_data_parallel_size（SPMD）

源码：`vllm/config/parallel.py:129`

```python
data_parallel_size_local: int = Field(default=1, ge=0)
"""Number of local data parallel groups. A value of 0 is a sentinel used by
the engine-args layer to signal that data parallelism was specified
externally (see `ParallelConfig.__post_init__`)."""
```

`data_parallel_size_local` 是 SPMD DP 的概念：同一进程跑多个 DP rank。

### 6.1 SPMD 模式

```
传统：每 DP rank 一个进程
   DP=4 → 4 个进程

SPMD：单进程跑多个 DP rank
   DP=4, DP_local=4 → 单进程跑 4 个 DP rank
   通过 async scheduling 在多个 DP rank 间切换
```

### 6.2 优势

- 进程间通信开销消失（共享内存）
- 内存共享（KV cache 可以共用部分）

### 6.3 现状

vLLM V1 探索中，不是默认模式。

## 7. 启动参数总结

| 参数 | 含义 |
|------|------|
| `--data-parallel-size N` | DP 副本数 |
| `--data-parallel-size-local M` | 单进程内 DP rank 数（SPMD） |
| `--data-parallel-rank R` | 当前 rank 的 DP 编号（external 模式） |
| `--data-parallel-master-ip IP` | DP master 的 IP（external 模式） |
| `--data-parallel-port P` | DP master 的端口 |
| `--distributed-executor-backend mp/ray` | 进程启动后端 |
| `--data-parallel-external-lb` | 启用 external_lb 模式 |
| `--data-parallel-hybrid-lb` | 启用 hybrid_lb 模式 |

## 8. 启动流程

### 8.1 简化版

```
1. vllm CLI 解析参数
2. 根据 data_parallel_backend:
   - mp: fork N 个子进程
   - ray: 起 N 个 actor
3. 每个进程:
   - 设置 VLLM_DP_RANK、VLLM_DP_WORLD_SIZE
   - 初始化 torch.distributed
   - 调用 initialize_model_parallel()
   - 创建 EngineCore + Worker
4. 所有进程 ready 后，开始服务
```

## 11. 选择启动模式的决策树

```
单节点部署？
├─ 是 → MP 模式（--data-parallel-size N）
└─ 否 → 跨节点
    ├─ MoE 模型？
    │   ├─ 是 → wide-EP，external_lb 或 hybrid_lb
    │   └─ 否 → Ray 模式（--distributed-executor-backend ray）
    └─ K8s 环境？
        ├─ 是 → external_lb + 每 rank 一个 pod
        └─ 否 → Ray 模式
```

---

## 12. 小结

- 三种启动模式：MP（默认）、Ray、External
- External 模式只支持 MoE（wide-EP 部署）
- Hybrid LB：节点内 vLLM LB + 节点间 external LB
- 关键参数：`--data-parallel-size`、`--data-parallel-external-lb`
